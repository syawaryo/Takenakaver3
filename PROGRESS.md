# 床スリーブ図 座標抽出デモ — 進捗と課題メモ

## 現在のアーキテクチャ

```
入力画像 → [並行処理]
  1. Azure DI OCR → 全テキスト + ピクセル座標
  2. OpenCV モルフォロジー → 通り芯の位置
  3. OpenCV HSV青フィルタ → 連結成分 → スリーブ青丸検出
  4. OpenCV テンプレートマッチング → 寸法接続点座標
→ SK番号の最近傍排他割り当て → スリーブとテキスト紐付け
→ Gemini VLM or 正規表現 → テキスト構造化パース
→ JSON + overlay PNG 出力
```

**Geminiの役割**: テキスト構造化パースのみ（`--no-vlm`で正規表現に置換可能）
**座標系**: 全てOpenCVピクセル座標で統一

## ファイル構成

```
src/
  models.py             — Pydanticデータモデル
  sleeve_detector.py    — HSV青フィルタ + 連結成分 + 円形度判定
  grid_detector.py      — モルフォロジー + 射影ヒストグラム
  ocr_extractor.py      — Azure DI OCR (座標はインチ→px変換)
  dimension_detector.py — テンプレートマッチング (Gemini画像生成は廃止)
  vlm_analyzer.py       — Gemini VLMテキスト解析 + regex fallback
  main.py               — パイプライン + CLI + overlay描画
```

## 現在の検出結果（スクリーンショットタブ削除後拡大.png）

- OCRテキスト: 125個（Φ/排水の後処理済み）
- 通り芯: 8本（V-7, V-182, V-478, V-914, V-1042, H-70, H-312, H-716）
- スリーブ: 11候補 → SK番号紐付き9個（SK-095〜097, 101〜104, 107, 108）
- 寸法接続点: 35個（テンプレートマッチング threshold=0.75, NMS=8px）
- スケール: 0.0574 px/mm

## 解決済みの課題

### 課題1: OCRがΦを¢と誤読する → ✅ 解決
- `ocr_extractor.py` に `_postprocess_ocr` を追加
- 置換ルール: `¢`→`Φ`、`##`→`排水`、`#:`→`排`
- OCR結果の全テキストに適用済み

### 課題2: 寸法接続点が連続する箇所で取りこぼす → ✅ 解決
- NMS距離を12px→8pxに縮小（`dimension_detector.py`）
- これにより近接する2点が統合されにくくなった

### 課題3: スリーブ青丸が全部取れない → ✅ 改善
- **方式A（従来）**: HSV青フィルタ + 連結成分 → 9個検出
- **方式B（新規）**: HoughCircles + 青色判定 + 内部明るさ判定 → 追加4個
- 統合・重複除去後: 11個（元10個から+1、SK紐付き9個に改善）
- HoughCirclesのフィルタ条件:
  - 円周上の青ピクセル比率 ≥ 34%
  - 内部明るさ ≥ 200（中空円＝白背景を要求）
  - param1=100, param2=30（厳格なエッジ検出）

## 未解決の課題

### 課題4: テキスト紐付けの精度
- SK番号→最近傍排他割り当ては機能している（8/10成功）
- 周辺テキスト（スペック）の収集半径120pxは場所によって過不足
- 複数スリーブのスペックテキストが近接していると混ざる
- **対策案**: 青の矢印（引出線）の方向を検出して、矢印の先のテキストだけを収集

### 課題5: 通り芯ラベルが取れない
- 通り芯端部の丸囲みラベル（X1, Y2等）がOCRで読めていない
- 低解像度のため文字が小さすぎる
- 現在は V-{position}, H-{position} で代替
- **対策案**: HoughCirclesで丸囲みを検出 → 近傍OCRテキスト

## 技術メモ

### Azure DI OCR座標変換
- Azure DIはインチ単位で返す（page.width/height がインチ）
- `scale_x = img_width_px / page_width_inch` で変換
- `content_type="image/png"`, body=`io.BytesIO(image_bytes)` で渡す

### HSV青フィルタの最適値
- H: 90-135（主要ピクセルはH=120-125）
- S: 50-255（スリーブ青はS=55-89程度、薄い）
- V: 50-255

### テンプレートマッチング
- template/image.png = 22x26px の寸法接続丸印
- cv2.TM_CCOEFF_NORMED, threshold=0.75
- マルチスケール: [0.5, 0.6, 0.7, 0.8, 0.9, 1.0, 1.1, 1.2, 1.4]
- NMS距離: 12px

### Gemini画像生成の教訓
- gemini-2.0-flash-exp-image-generation で丸印検出を試みた
- **結果**: 出力画像の解像度が元画像と異なる（1646x756→1024x469）
- 寸法丸印だけでなく壁・柱の輪郭まで赤く塗る → 精度壊滅
- プロンプトで限定しても改善せず → **この用途には使えない**と判断
- テンプレートマッチングに切り替えて解決

### venv
- WSL (Linux) 用: `.venv/` （uv で作成, Python 3.12.3）
- Windows MinGW/PowerShell からは使えない → WSLから実行すること
